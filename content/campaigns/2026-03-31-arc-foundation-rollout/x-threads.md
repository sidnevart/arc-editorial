# X Threads

## Thread 1. Почему одной сильной модели недостаточно

1. У большинства AI products проблема не в том, что модель слабая.
2. Проблема в том, что вокруг неё нет runtime.
3. Нет context boundaries.
4. Нет policy layer.
5. Нет budget control.
6. Нет артефактов, по которым потом можно понять, что вообще произошло.
7. Поэтому ARC мы строим не вместо Codex, а поверх него.
8. Codex даёт reasoning. ARC даёт управляемую среду.

## Thread 2. Локальные миниаппы из чата

1. Один из самых сильных UX-моментов в AI — когда ответ перестаёт быть текстом.
2. Попросил симуляцию.
3. Получил работающий миниапп.
4. Закрыл.
5. Вернулся к старому чату.
6. Нажал “Запустить снова”.
7. И он опять жив.
8. Вот это уже не чатик. Это runtime.

## Thread 3. Context tool как superpower

1. Контекст — это не “засунуть больше текста в prompt”.
2. Контекст — это управление вниманием модели.
3. Хороший context tool должен уметь:
4. найти кандидатов
5. отобрать релевантное
6. объяснить выбор
7. сравнить baseline vs optimized
8. и не забыть про бюджет

## Thread 4. Почему multi-agent системы разваливаются

1. Пока агент один, почти всё работает “и так”.
2. Потом появляется второй, третий, десятый.
3. И внезапно выясняется, что главный вопрос — не “каких агентов ещё сделать”.
4. А “кто чем владеет”.
5. Hooks.
6. Memory scopes.
7. Runtime permissions.
8. Без environment rules всё это быстро становится кашей.

## Thread 5. Экономия токенов без потери качества

1. Умная экономия токенов — это не “отрезать всё”.
2. Это:
3. не слать шум
4. отправлять часть задач локально
5. минимизировать provider-facing context
6. включать дорогой путь только когда он реально нужен
7. поэтому budget layer — это часть качества системы
8. а не просто бухгалтерия
