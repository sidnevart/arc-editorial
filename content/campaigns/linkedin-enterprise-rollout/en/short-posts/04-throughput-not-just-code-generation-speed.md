# Post 04: the real metric is throughput, not typing speed

## Copy

I think many teams still measure AI tooling in engineering too narrowly.

The question is not “does it write code faster?”
The question is “what does it do to team throughput?”

A good runtime affects much more than code generation:

- faster context acquisition
- lower cost of entering an unfamiliar part of the codebase
- more structured execution
- verify/review/docs becoming part of the normal flow

That is why I prefer talking about delivery throughput instead of “2x faster coding”.

The real value is reducing engineering friction and making task execution more structured and repeatable.
