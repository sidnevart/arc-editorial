# LinkedIn Post: как мы получили до 2x экономии токенов

## Хук

Один из самых недооценённых вопросов в AI tooling для разработки: сколько шума вы таскаете в prompt.

## Текст поста

Один из сильных эффектов, который мы увидели после перехода от статического контекста к query-driven packaging, это заметное снижение token waste.

По внутренним benchmark-сценариям получилось до 2x экономии токенов на типовых инженерных задачах.

Почему это произошло:

- перестали тащить в модель весь контекст “на всякий случай”
- отделили human-authored maps от machine-generated index
- начали вытаскивать только релевантные docs, code surfaces и memory
- добавили дешёвый retrieval через `rg` и optional `ast-grep`

Важный момент:

экономия токенов здесь сама по себе не главная метрика.

Главное то, что вместе с этим обычно растёт и качество:

- меньше шумных ответов
- меньше ложных допущений
- меньше ситуаций, когда модель уверенно говорит не по делу

То есть цель не “сделать prompt меньше”.
Цель — сделать контекст точнее.

И это хорошо чувствуется в production-среде, где стоимость ошибки и стоимость токенов обе реальные.

## Формулировка для осторожной публикации

Если не хочется звучать как маркетинг, я бы писал так:

`В ряде внутренних benchmark-задач переход на query-driven context packaging дал до 2x снижения token consumption без ухудшения качества ответа.`

Это звучит сильнее и честнее, чем “мы всегда экономим 50% токенов”.
