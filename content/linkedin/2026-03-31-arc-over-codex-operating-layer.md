# LinkedIn: ARC над Codex — это не обвёртка, а operating layer

Большая ошибка AI tooling сегодня — строить продукт так, будто главная задача уже решена, если у тебя есть доступ к сильной модели.

Нет, не решена.

Сильная модель сама по себе не даёт:

- управляемый контекст
- policy-driven поведение
- бюджетный контроль
- repeatable artifacts
- multi-agent environment

Именно поэтому мы строим ARC не вместо Codex, а поверх него.

Codex даёт reasoning.
ARC даёт runtime.

Для меня это и есть сильная продуктовая позиция: не спорить с моделью там, где она сильна, а закрывать системный слой, без которого AI-продукт остаётся дорогим и хрупким.

Сильная модель + слабый runtime = красивое демо.

Сильная модель + сильный runtime = уже инженерная система.
