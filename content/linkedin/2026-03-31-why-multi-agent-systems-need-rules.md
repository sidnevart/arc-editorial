# LinkedIn: multi-agent среда ломается без правил раньше, чем вам кажется

Почти любая multi-agent система выглядит красиво на старте.

Проблемы начинаются позже:

- появляются разные доменные агенты
- каждому хочется свои hooks
- кто-то трогает memory
- кто-то просит больше runtime permissions
- слой правил начинает течь

И в этот момент выясняется, что главный вопрос — не “каких агентов мы добавим дальше”, а “какие границы мы им поставили”.

Поэтому preset environment rules в ARC для меня — это не документация про расширения.

Это инфраструктура выживания платформы.

Если у тебя нет жёсткой environment model, то рост числа агентов почти гарантированно превращается в рост хаоса.
