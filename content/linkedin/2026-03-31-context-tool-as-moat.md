# LinkedIn: context tool становится moat’ом над моделью

Сильная модель сегодня уже не дефицит сама по себе.

Настоящее преимущество всё чаще появляется в другом месте: кто лучше управляет контекстом вокруг неё.

Именно поэтому мне кажется важным строить `ctx` как отдельный tool boundary, а не как внутренний helper.

Когда context system умеет:

- индексировать репозиторий
- держать memory отдельно
- собирать query-driven context packs
- сравнивать baseline vs optimized
- объяснять selection через provenance и quality signals

ты получаешь не “подсобную фичу”, а настоящий слой продукта.

ARC для меня именно про это: Codex остаётся reasoning engine, а `ctx` становится той частью системы, которая помогает этому engine тратить интеллект туда, где он нужен.
