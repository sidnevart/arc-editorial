# Post 03: до 2x снижения token waste

## Текст

Один из самых интересных эффектов после перехода от статического context pack к query-driven retrieval оказался довольно прикладным:

в ряде внутренних benchmark-сценариев мы увидели до 2x снижения token consumption на типовых инженерных задачах.

Что поменяли:

- перестали тащить в prompt всё подряд
- разделили human-authored maps и machine-generated index
- начали поднимать только релевантные docs, code surfaces и memory
- добавили дешёвый retrieval через `rg` и optional `ast-grep`

Но важнее другое.

Экономия токенов здесь не самоцель.
Хороший эффект в том, что вместе с ней обычно снижается и шум:

- меньше ложных допущений
- меньше нерелевантных ответов
- меньше уверенности “не по делу”

То есть цель не “сделать prompt меньше”.
Цель — сделать контекст точнее.

Если интересно, следующим постом разберу, почему это уже влияет не только на стоимость, но и на throughput команды.

## Промпт для картинки

Сделай визуализацию для LinkedIn о снижении token waste в AI developer tooling.
Покажи два контекста: раздутый шумный prompt и компактный релевантный context pack со слоями task, docs, code, memory, search hits.
Стиль: зрелая инженерная инфографика, светлый фон, тёмно-зелёные, графитовые и синие акценты.
Формат: вертикальный 4:5.
