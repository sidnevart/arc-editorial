# Post 09: deterministic pipeline лучше хаотичного agent loop

## Текст

С AI очень легко построить систему, которая выглядит умной, но плохо управляется.

Особенно если всё держится на одном бесконечном agent loop, где модель сама решает, что делать дальше.

Для демо это может выглядеть впечатляюще.
Для реальной инженерной работы это часто слабая конструкция.

Поэтому мне сильно ближе идея deterministic pipeline:

- plan
- run
- verify
- review
- docs

Почему:

- проще наблюдать состояние
- легче локализовать ошибки
- проще вводить правила и approvals
- легче объяснить поведение пользователю
- проще масштабировать на организацию

Автономия важна.
Но автономия без структуры слишком быстро превращается в хаос с дорогим токеновым хвостом.
