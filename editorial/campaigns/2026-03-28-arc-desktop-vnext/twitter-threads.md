# X / Twitter thread ideas

## Thread 1. Why “AI IDE” is often the wrong framing

1. We kept saying “AI IDE”, but the real value was not code editing.
2. The useful part was a workspace where a conversation becomes a plan, demo, document, or simulation.
3. Files matter, but only as secondary evidence and inspect surfaces.
4. Tasks matter, but users should not need to babysit a task manager.
5. Sessions turned out to be the right top-level unit.
6. That changed the entire product direction.

---

## Thread 2. `Study / Work / Hero` should not be cosmetic

1. A lot of tools offer “modes”.
2. Most of them are just different prompts.
3. Users expect much more than that.
4. `Study` should not silently do the work for you.
5. `Work` should not turn into hidden autonomy without permission.
6. `Hero` should be autonomous, but still bounded and inspectable.
7. If the environment does not enforce it, the mode is not real.

---

## Thread 3. Embedded demos beat raw localhost links

1. If an agent builds a mini app, the user wants to see it immediately.
2. “Here is a localhost URL” is not a product experience.
3. The right default is embedded preview.
4. External open is still useful, but only as a secondary action.
5. Runtime lifecycle control matters just as much as generation.

---

## Thread 4. Verification agents caught real product bugs

1. We added verification agents before the next feature pass.
2. They immediately found stale scaffold state in `.arc`.
3. They found TODO provider guidance.
4. They found missing skills.
5. They found session routing mistakes.
6. That saved us from building on top of bad assumptions.
